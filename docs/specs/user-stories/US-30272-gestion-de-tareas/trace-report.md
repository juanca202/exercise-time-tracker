## Observaciones y pendientes

- AC-002 / AC-010: agregar, análogo a `projectsStore.test.ts`, un test de round-trip explícito (`localStorage.setItem` con datos de `tasksStore` → `rehydrate()` → assert) para dejar la persistencia de Tareas y Registros de Tiempo manuales tan explícitamente verificada como la de Proyectos. No es un hueco funcional (el mecanismo `persist` es el mismo y ya está probado a nivel de `projectsStore`), pero cierra la asimetría de cobertura entre ambos stores.
- AC-012 / AC-013: siguen sin harness de medición de latencia, tal como documentan sus propios TC. Si se requiere verificación automatizada real (no solo funcional), habría que construir el harness con `performance.now()` + tolerancia que ambos TC piden explícitamente.
